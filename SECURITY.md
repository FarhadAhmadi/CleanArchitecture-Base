# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch.

## Reporting a Vulnerability

1. Do not open a public issue for security vulnerabilities.
2. Send a private report to: `security@cleanarchitecture.local`
3. Include:
- affected endpoint/module
- reproduction steps
- impact assessment
- suggested remediation (if available)

## Response Targets

- Initial acknowledgment: within 48 hours
- Triage decision: within 5 business days
- Fix timeline:
- Critical/High: as soon as possible with prioritized hotfix
- Medium/Low: next planned release cycle

## Coordinated Disclosure

After a fix is released, coordinated disclosure is welcomed. Please avoid publishing exploit details before the patch is available.
